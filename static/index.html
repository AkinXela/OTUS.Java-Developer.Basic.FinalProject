1. Описание проекта

Данный проект - серверное Java Core приложение.

Приложение генерирует поддельные (фейковые) данные и отдает их по API.

В качестве фейковых данных взяты следующие сущности:

  • First name
  • Last name
  • Address
  • Job position
  • Hobby

Фейковые данные генерируются на лету, по запросу.

Факт каждого запроса записывается в базу данных, будь то он ошибочный или успешный.

2. Использование приложения

API проекта разбито на 2 части:

  • API фейковых данных
  • API статистики

Все тестовые API запросы расположены в папке проекта requests в корне проекта.


2.1 API фейковых данных

GET {{url}}/ - запрос возвращает в теле ответа файл index.html (данный файл).

GET {{url}}/import.this - запрос возвращает в теле ответа файл import.this.

GET {{url}}/fakeitems - запрос возвращает сгенерированные фейковые данные. Начальное количество генерации - 10.

GET {{url}}/fakeitems?quantity=number - запрос возвращает сгенерированные фейковые данные.

number - число записей фейковых данных с начала, которые нужно вывести. Если данный меньше, чем number, выводятся все данные.

GET {{url}}/fakeitem?id=number - запрос возвращает одну запись сгенерированных фейковых данных.

number - id записи в репозитории

POST {{url}}/fakeitems - запрос создает новую фейковую запись и добавляет ее в репозиторий.

Тело запроса должно быть заполнено. В теле может быть как одна запись, так и несколько.

POST {{url}}/fakeitems?quantity=number - запрос создает новую фейковую запись. Тело запроса может быть пустым.

number - количество новых фейковых данных, которое необходимо сгенерировать и добавить в репозиторий

PUT {{url}}/fakeitems - запрос изменяет фейковую запись. Тело запроса должно быть заполнено.

В теле может быть как одна запись, так и несколько.

DELETE {{url}}/fakeitems?id=number - удаляет фейковую запись из репозитория

number - id записи в репозитории

OPTIONS {{url}} - запрос возвращает заголовок, в котором перечислены разрешенные методы.


2.2 API статистики

GET {{url}}/statistics - запрос возвращает все записи статистики из базы данных

GET {{url}}/statistics?method=word - запрос возвращает записи статистики из базы данных, отфильтрованные по методу.

word - метод в заголовке запроса, например GET

GET {{url}}/statistics?id=number - запрос возвращает запись статистики из базы данных, отфильтрованную по id.

number - id записи статистики в базе данных

PUT {{url}}/statistics - запрос изменяет поле 'billed' фейковой записи. Все остальные данные изменить нельзя.

Тело запроса должно быть заполнено. В теле может быть как одна запись, так и несколько.

DELETE {{url}}/statistics?id=number - удаляет запись статистики из базы данных

number - id записи в репозитории
